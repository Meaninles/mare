# 多资产库迁移与回归清单

## 1. 目标

本次改造的目标不是在原有 `backend/data/mam.db` 之上继续堆功能，而是把系统正式切换为：

- 应用层只负责资产库入口、最近打开、当前活动库、全局偏好。
- 一个资产库对应一个独立的 SQLite 文件，默认后缀为 `.maredb`。
- 资产、存储节点、导入规则、同步任务、备份导入导出，都属于当前资产库内部状态。
- 未打开资产库前，不允许直接进入资产页，也不允许执行库级扫描、导入、同步、备份操作。
- 旧单库 `mam.db` 需要迁移为一个默认资产库，同时保留原始旧库，保证可回退。

## 2. 旧单库迁移方案

### 2.1 入口

- Welcome 页检测旧单库 `mam.db`。
- 如果检测到旧库且未完成迁移，则展示“迁移旧单库为默认资产库”操作。
- 如果目标资产库文件已存在，则展示“打开已迁移的默认资产库”操作。

### 2.2 迁移行为

- 源文件：现有 `backend/data/mam.db`。
- 默认目标文件：同目录下的 `default-library.maredb`。
- 迁移时优先使用 SQLite 快照复制；失败时退回文件复制。
- 迁移后会对目标库执行库内 migration，并补齐 `library_metadata`。
- 系统会对以下计数做一致性校验：
  - `assets`
  - `replicas`
  - `storage_endpoints`
  - `import_rules`
  - `tasks`
- 校验通过后才会正式提升为目标资产库文件。
- 系统会生成一份迁移清单：`default-library.migration.json`。

### 2.3 回退边界

- 原始 `mam.db` 不删除、不覆盖。
- 迁移失败时不应污染原始旧库。
- 目标库只在校验成功后才作为当前资产库打开。
- 如果目标库已存在，优先走“直接打开已迁移资产库”，不重复覆盖。

## 3. 当前实现边界

### 3.1 已完成

- 应用壳数据库已支持资产库注册、最近打开、当前活动库。
- Go 后端已支持空载启动、按需打开/关闭当前资产库。
- Welcome 页已成为正式入口。
- 存储节点、扫描、导入、同步、任务、库级备份导入导出，均要求存在当前资产库。
- 旧单库迁移 API 已提供：
  - `GET /api/v1/libraries/legacy/status`
  - `POST /api/v1/libraries/legacy/migrate`

### 3.2 当前约束

- 第一阶段只支持单窗口单资产库。
- 旧单库迁移默认只处理本地 legacy sqlite 文件。
- 当前“统一文件资产”仍以资产库内标准逻辑路径为主键语义，多端副本围绕该逻辑路径展开。

## 4. 最小回归清单

### 4.1 Welcome 与资产库入口

- 启动应用时，未打开资产库时应停留在 Welcome 页。
- 可以新建 `.maredb` 资产库并自动打开。
- 可以手动打开已有 `.maredb` 资产库并登记到最近列表。
- 最近使用列表可重新打开任一已登记资产库。
- 关闭当前资产库后，应回到未挂载状态。

### 4.2 旧单库迁移

- 当 `backend/data/mam.db` 存在时，Welcome 页应出现迁移提示。
- 迁移完成后应自动打开新资产库。
- 新资产库应被登记到最近使用列表。
- 原始 `mam.db` 仍然保留。
- 应生成对应的 `.migration.json` 清单文件。

### 4.3 库内数据边界

- 未打开资产库时，访问资产、存储、导入、同步、任务相关接口应返回“库未打开”错误。
- 打开 A 库后只能看到 A 库的资产、端点、任务。
- 切换到 B 库后，前端 Query 缓存不能继续显示 A 库数据。
- 设置页中的备份、导入、校验提示必须明确作用于当前资产库。

### 4.4 存储节点与扫描

- 新建端点只写入当前资产库，不写入应用壳数据库。
- 全量扫描和单端点扫描都只作用于当前资产库。
- `IMPORT_SOURCE` 端点不能混入正式受管扫描。
- 扫描产出的任务记录只能出现在当前资产库任务列表中。

### 4.5 导入、同步与副本

- 导入规则只对当前资产库生效。
- 资产恢复和批量恢复应写回当前资产库的副本表。
- 恢复目标路径应按资产标准逻辑路径解析，而不是跟随某个副本的偶然物理路径。
- 多端“打开所在文件夹”能力应基于当前资产的标准逻辑路径推导各端目录。

### 4.6 库级备份与恢复

- 设置页导出备份时，应只导出当前资产库的端点、规则、资产快照。
- 导入备份时，应只写回当前资产库。
- 导入带资产快照的备份后，应可以立即看到资产视图，然后再执行端点校验。

## 5. 建议的人工验收顺序

1. 启动应用，确认 Welcome 页为默认入口。
2. 检查是否出现旧单库迁移卡片。
3. 执行迁移，确认自动进入默认资产库。
4. 检查存储节点、资产视图、同步页、任务页是否能正常加载。
5. 导出一次当前资产库备份，再执行一次导入恢复。
6. 切换到另一个空白资产库，确认不会看到前一个库的数据。
7. 关闭资产库，再确认库级页面和库级操作被正确阻止。
