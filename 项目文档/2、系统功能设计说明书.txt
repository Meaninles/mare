Mare 系统功能设计说明书

一、文档目的

本说明书用于描述 Mare 当前项目的主要业务功能、核心对象、交互入口与后续扩展方向。本文档以当前代码事实为准。

二、系统概述

Mare 是一个桌面端多媒体资产管理客户端，围绕“统一资产视图”展开。用户不再直接面对多个孤立的存储目录，而是通过资产库统一查看同一逻辑资产在多个端点上的副本状态。

当前系统主体由以下功能域组成：

1. 资产库
2. 资产详情
3. 同步中心
4. 导入中心
5. 存储管理
6. 设置
7. 后台扫描与媒体处理

三、核心业务对象

1. 存储端点
   表示一个被纳管的存储位置，例如本地目录、QNAP、115、移动硬盘等。

2. 资产
   表示用户视角看到的统一资源。一个资产通常由 logical_path_key 归并得到。

3. 副本
   表示某个资产在某个端点上的一个实际文件实体。

4. 副本版本
   表示副本对应的尺寸、时间戳、校验相关信息，是差异分析和后续同步的基础。

5. 后台任务
   表示扫描、缩略图、视频封面、音频元数据等后台执行单元。

6. 媒体派生物
   表示由原始媒体文件生成的缩略图、视频封面、音频封面等缓存文件。

四、功能模块说明

4.1 资产库

资产库是系统主入口，当前已经接入本地 Catalog 数据。

当前要求与已实现方向：

1. 一个资产只显示一条。
2. 资产项展示名称、媒体类型、时间、资产状态和副本状态标签。
3. 支持轻量筛选：
   全部、图片、视频、音频。
4. 支持状态筛选：
   全部、完整可用、部分缺失、仅单端存在、全部缺失。
5. 支持时间排序：
   最新优先、最早优先。
6. 支持分页浏览。
7. 支持空状态和加载状态。
8. 当前已支持顶部搜索词对列表进行联动过滤。

4.2 资产详情

资产详情页是资产操作入口，当前以“浏览 + 预留操作”为主。

当前要求与已实现方向：

1. 展示资产名称、逻辑路径、媒体类型、时间和状态。
2. 展示所有副本列表。
3. 展示端点名称、物理路径、副本状态、大小、时间等信息。
4. 预留恢复到某端、删除某端副本、打开所在位置、重新扫描等操作位。
5. 集成媒体预览。

4.3 媒体预览

当前已实现基础预览闭环：

1. 图片支持预览与放大显示。
2. 视频支持基础播放与暂停。
3. 音频支持基础播放、暂停和进度显示。
4. 后端提供 poster 与 preview 资源接口。

4.4 同步中心

同步中心当前为结构入口，后续用于承载：

1. 可恢复资产列表
2. 缺失副本列表
3. 同步中任务
4. 失败任务
5. 冲突候选项

当前更多承担页面占位与后续扩展承接作用。

4.5 导入中心

导入中心当前也属于后续重点功能入口，目标是支持可移动设备作为导入源时的浏览、筛选、规则分发与导入任务执行。

4.6 存储管理

存储管理用于展示和管理已纳管的存储端点。

当前后端已具备的基础能力包括：

1. 端点注册
2. 端点列表读取
3. 全量扫描
4. 单端点重扫

后续会继续补充端点编辑、重绑定、健康检查可视化与角色管理。

4.7 设置

当前设置页已经具备客户端层面的主题切换能力。

当前已接入：

1. 浅色主题
2. 深色主题
3. 桌面客户端统一偏好入口

后续将继续接入：

1. 窗口偏好
2. 导入规则偏好
3. 端点重绑定
4. 配置导入导出

五、后台功能说明

5.1 扫描与 Catalog 建立

系统通过扫描端点下的媒体文件，将结果归并为资产与副本结构。

当前实现方向：

1. 支持全量扫描。
2. 支持单端点重扫。
3. 扫描结果写入资产、副本、副本版本。
4. 对派生媒体任务进行异步排队。

5.2 图片缩略图

当前已具备：

1. 生成固定尺寸缩略图。
2. 按资产 ID 缓存到本地目录。
3. 写入数据库中的媒体预览记录。
4. 二次访问优先读取缓存。

5.3 视频封面

当前已具备：

1. 基于 FFmpeg 提取视频封面。
2. 将封面缓存到本地目录。
3. 与资产主缩略图引用关联。

5.4 音频元数据

当前已具备：

1. 提取时长、编码、采样率、声道等基础信息。
2. 读取内嵌封面并落到媒体预览记录。
3. 将结果写回本地数据库。

5.5 后台任务

当前任务系统主要支撑：

1. 缩略图生成
2. 视频封面提取
3. 音频元数据提取
4. 后续也可扩展到扫描、同步、导入、转写等任务

六、非功能目标

1. 界面简洁、克制、统一，适合桌面端长时间使用。
2. 保持主题一致性，浅色和深色仅切换配色，不改变交互结构。
3. 任务失败不影响资产基础可见性。
4. 尽量保证扫描、预览、媒体处理链路的可解释性。
5. 为后续复杂同步、恢复、搜索能力保留扩展空间。

七、当前范围说明

本文档中的“当前已具备”表示代码层面已有基础实现。
本文档中的“后续将继续接入”表示该能力已纳入当前项目方向，但尚未完全闭环。
